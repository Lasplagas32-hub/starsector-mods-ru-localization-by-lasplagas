# Начать отсюда - RU MorphLib для переводчика

Этот файл объясняет, какой слой брать под конкретную задачу.

## Главные правила

1. Не склеивайте русские окончания с токенами вслепую. Лучше берите готовый падежный токен.
2. Для NPC используйте `$rulm_*`, для игрока - `$rulp_*`.
3. Статичные числа переводятся руками. Для `5 credits` не нужен suffix-layer.
4. Dynamic suffix нужен только когда число берётся из memory, ресурса игрока или API.
5. Если мод уже даёт красивую цену вроде `$coff_bribeprice_text`, оставляйте её для числа и добавляйте только нужную русскую форму рядом.
6. RU MorphLib не переводит строки автоматически. Он только подставляет правильные формы в уже переведённый текст.
7. Не используйте `$ruln_suf_num_*`: это не поддерживаемый пользовательский workflow.
8. Не ждите угадывания неизвестных чисел из чужого мода. Сначала нужен per-mod audit.


## Выбор слоя

| Задача переводчика | Слой | Пример |
|---|---|---|
| Текущий NPC, пленник, офицер, контакт, собеседник | `$rulm_*` | `$rulm_thisNom $rulm_prisonerNom` |
| Игрок и выбранное обращение `Сэр/Мэм/Капитан` | `$rulp_*` | `$rulp_Address, $rulp_youNom $rulp_arrivedDirect` |
| Полное слово после числа прямо в токене | `$ruln_num_*` | `$ruln_num_5_credit` -> `кредитов` |
| Полное слово для ресурса игрока | `$ruln_player*` | `$ruln_playerCreditsValue $ruln_playerCredits_credit` |
| Полное слово для числа из memory/dialogue variable | `$ruln_mem_*` | `$coff_ransomprice_text $ruln_mem_coff_ransomprice_credit` |
| Только окончание/хвост слова для динамического числа | `$ruln_suf_mem_*`, `$ruln_suf_player_*` | `$price_text кредит$ruln_suf_mem_price__blank__a__ov` |
| Java/Kotlin/JAR/UI-код | `RuMorphQuantity` | `RuMorphQuantity.word(amount, "credit")` |


## Мини-рецепты

### 1. Текущий пленник или NPC

```text
$rulm_thisNom $rulm_prisonerNom сейчас $rulm_capturedState.
Говорить $rulm_withHim.
```

Результат:

```text
этот пленник сейчас захвачен. Говорить с ним.
эта пленница сейчас захвачена. Говорить с ней.
```

### 2. Не повторять слово пленник

```text
$rulm_GuyOrGirlNom молча смотрит в пол. Конвой ведёт $rulm_himAcc к шаттлу.
```

Результат:

```text
Парень молча смотрит в пол. Конвой ведёт его к шаттлу.
Девушка молча смотрит в пол. Конвой ведёт её к шаттлу.
```


### 3. Н-форма после уже написанного предлога

Если предлог уже есть в строке, не используйте `$rulm_himAcc` или `$rulm_himGen` после него. Возьмите `$rulm_himAfterPrep*`.

```text
Никто не вступится за $rulm_himAfterPrepAcc.
Поговорить с $rulm_himAfterPrepIns.
```

Результат:

```text
Никто не вступится за него. / Никто не вступится за неё.
Поговорить с ним. / Поговорить с ней.
```

### 4. Обращение к игроку

```text
$rulp_Address, $rulp_youNom $rulp_arrivedDirect вовремя.
```

Результат:

```text
Сэр, вы прибыли вовремя.
Мэм, вы прибыли вовремя.
Капитан, вы прибыли вовремя.
```

### 5. Число из memory

Если мод уже показывает цену как `$coff_bribeprice_text`, оставьте её для числа:

```text
Заплатить $coff_bribeprice_text $ruln_mem_coff_bribeprice_credit.
```

Пример вывода:

```text
Заплатить 9,375¢ кредитов.
```

## Что не делать

Плохо:

```text
$rulm_prisonerNomа
```

Так можно получить мусор. Лучше взять готовую форму:

```text
$rulm_prisonerAcc
```

Плохо:

```text
5 $ruln_suf_num_5__blank__a__ov
```

Статичное число переводится руками:

```text
5 кредитов
```
