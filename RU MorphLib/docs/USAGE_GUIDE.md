# RU MorphLib - инструкция для переводчика

## 1. Что делает мод

RU MorphLib добавляет в Starsector расширенные русские токены `$rulm_*`.

Они помогают переводить диалоги с персонажами разного пола без обезличивания и кривой грамматики.

Пример:

```text
$rulm_He поднял$rulm_past взгляд.
Поговорить $rulm_withHim.
```

Результат:

```text
Он поднял взгляд. Поговорить с ним.
Она подняла взгляд. Поговорить с ней.
```

## 2. Как подключить к переводу

### Шаг 1

Игрок должен установить и включить `RU MorphLib`.

### Шаг 2

В переводимом моде нужно добавить зависимость в `mod_info.json`:

```json
{
  "id": "ru_morphlib",
  "name": "RU MorphLib"
}
```

Если у мода уже есть `dependencies`, добавь RU MorphLib в существующий список.

### Шаг 3

В `rules.csv` можно использовать `$rulm_*` токены.

## 3. Где использовать

Ожидаемо работает:

```text
rules.csv text
rules.csv options
AddText
SetTooltip
AddOption / SetOption, если они проходят через стандартный token replacement
```

Пример `text`:

```text
Собеседни$rulm_nikNom попытал$rulm_pastReflexive сохранить спокойствие.
```

Пример `options`:

```text
AskMore:Поговорить $rulm_withHim.
Release:Освободить пленни$rulm_nikAcc.
```

Пример `script-visible`:

```text
AddText "$rulm_He выглядел$rulm_past ранен$rulm_shortAdj, но держал$rulm_past себя в руках."
SetTooltip optionId "Поговорить $rulm_withHim."
```

## 4. Где нужна проверка

Может не работать без дополнительной интеграции:

```text
кастомный hardcoded UI в JAR
самописные панели
кастомные магазины
кастомные intel-экраны
строки из JSON/CSV, если мод выводит их напрямую без RulesAPI
LunaLib settings
```

Если после установки RU MorphLib на экране видно сырой `$rulm_*`, значит конкретная строка не проходит через стандартную подстановку токенов.

## 5. Важное правило про контекст

`$rulm_*` обычно относится к текущему `activePerson` / персонажу диалога.

Это не обязательно игрок.

Не используй NPC-токены для обращения к игроку, пока отдельный player address layer не реализован.

## 6. Как писать морфемные токены

Морфемные токены пишутся без пробела с основой слова:

```text
пленни$rulm_nikNom
пленни$rulm_nikAcc
поднял$rulm_past
попытал$rulm_pastReflexive
пленн$rulm_adjAccAnim
```

Результат:

```text
пленник / пленница
пленника / пленницу
поднял / подняла
попытался / попыталась
пленного / пленную
```

## 7. Как писать готовые формы

Готовые формы обычно пишутся как отдельное слово или фраза:

```text
Поговорить $rulm_withHim.
Стоять перед $rulm_himIns.
Это $rulm_whichNom говорил$rulm_past первым.
```

## 8. Старые токены Starsector

RU MorphLib не ломает и не заменяет старые токены.

Можно смешивать:

```text
$personName посмотрел$rulm_past на вас. Поговорить $rulm_withHim.
```

Старые токены вроде `$playerName`, `$heOrShe`, `$hisOrHer`, `$himOrHer` продолжают работать как раньше.

В RU MorphLib есть дубли старых gender-токенов в нашем namespace:

```text
$rulm_heOrShe
$rulm_hisOrHer
$rulm_himOrHer
```


## 10. Можно ли ставить токены подряд

Да, `$rulm_*` токены можно писать подряд без пробела, если ты точно понимаешь, какой результат должен получиться.

Технически это допустимо:

```text
$rulm_tokenOne$rulm_tokenTwo
```

Игра просто заменит каждый токен на его `masc` или `fem` значение.

Но RU MorphLib не проверяет грамматику сочетания. Если соединить несовместимые токены, получится неправильная форма.

Плохой пример:

```text
пленни$rulm_nikAcc$rulm_adjAccAnim
```

`$rulm_nikAcc` уже содержит весь хвост `ка / цу`, поэтому добавлять к нему окончание прилагательного нельзя.

Рекомендуемый обычный вариант:

```text
пленни$rulm_nikAcc
поднял$rulm_past
пленн$rulm_adjAccAnim
```

Результат:

```text
пленника / пленницу
поднял / подняла
пленного / пленную
```

Практическое правило:

```text
основа + один морфемный токен
```

или:

```text
готовая форма $rulm_*
```

Два и более токена подряд использовать только осознанно, когда формы действительно грамматически совместимы.

## 11. Совместимость версий

Если перевод использует RU MorphLib, укажи минимальную версию:

```text
Требуется RU MorphLib 0.5.4 или новее.
```

Новые версии должны добавлять токены без удаления старых. Старые переводы не должны ломаться.
